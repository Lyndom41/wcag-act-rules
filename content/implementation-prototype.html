Axe-core Test Rule Implementation

Implementation Summary

Axe-core version 4.3.0 was published on 1 December 2021 by Deque Systems. This implementation tests for WCAG 2.1 (and older), level A and AA. Unless otherwise indicated, results are fully automatic. Test data was last updated 24 January 2022.

Complete Consistency (2)

Completely consistent implementations consider all aspects of a rule's applicability and expectations.

WCAG Test Rule Coverage
1.3.1 ARIA required context role Automatic (8/10)
1.3.1 Element with presentational children has no focusable content Automatic (20/20)

Partial Consistency (5)

Partially consistent implementations consider some, but not all aspects of a rule's applicability and expectations.

WCAGTest RuleCoverage

Minimal Consistency (2)

Minimally consistent implementations consider some or all aspects of a rule's applicability, without considering its expectations.

WCAGTest RuleCoverage

Test Description

ACT rules are tested using a canary version of axe-core, which can be accessed by installing `axe-core@next` from NPM. For these results, all rules including “experimental” rules are enabled. Checks with “unsupported” in its ID are disabled, which deal with issues of accessibility support issues which ACT rules do not account for, often by design.
